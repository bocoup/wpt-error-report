wpt-error-report

The following results represent a consolidated report of web-platform-tests (WPT) test files that fail to run to completion across all target browsers. These incomplete test runs cause inconsistent test completion counts when running the WPT test suite.

How can you help

  1. Find the offending test file you'd like to tackle in the list below.
  2. Fork the w3c/web-platform-tests repository.
  3. Follow the steps at w3c/web-platform-tests#setting-up-the-repo.
  4. Open your selected file in a browser that it is shown to be failing below.
  5. Figure out why the tests in the file do not run to completion.
  6. Fix that problem, commit your work in a branch of on your fork of WPT, and open a pull request.
  7. Here are some friendly people who you can tag as a reviewer on your pull request to get some extra support:

Analysis

The most likely reason that a test does not run to completion is a JavaScript error that's not handled correctly. This can manifest in myriad ways. Here are some helpful starting points for determining the cause of an incomplete test run in one of the test files:

  • Is there an assumption about an API method that's not yet fully implemented?
  • Is there an assumption about an API surface property that's not yet fully implemented?
  • Is it an async test that does not resolve before the test harness times out?
